Test Results
  • Windows 10/NVDA 2018.41/Firefox 60.7.2esr - heading is announced as a description when reading menu-items with the arrow keys
  • Windows 10/NVDA 2018.41/Chrome 75.0.3770.100 - heading is announced as a description when reading menu-items with the arrow keys
  • Windows 10/NVDA 2018.41/IE11 - heading is announced as a description when reading menu-items with the arrow keys
  • Windows 10/NVDA 2018.41/Edge - heading is announced as a description when reading menu-items with the arrow keys
  • Windows 10/JAWS 2019/Firefox 60.7.2esr - heading is announced as a description after the accessible name and keyboard usage instructions when reading menu-items with the arrow keys
  • Windows 10/JAWS 2019/Chrome 75.0.3770.100 - heading is announced as a description after the accessible name and keyboard usage instructions when reading menu-items with the arrow keys
  • Windows 10/JAWS 2019/IE11 - heading is announced as a description after the accessible name and keyboard usage instructions when reading menu-items with the arrow keys
  • Windows 10/JAWS 2019/Edge - heading is NOT announced as a description after the accessible name and keyboard usage instructions when . Additionally, it is not possible to read the heading with the virtual cursor
  • Mac OS X 10_14_4/VoiceOver/Safari Version 12.1.1 (14607.2.6.1.1) - heading is announced as a description after the accessible name when reading menu-items with the arrow keys
  • Mac OS X 10_14_4/VoiceOver/Chrome Version 75.0.3770.100 - heading is announced as a description after the accessible name when reading menu-items with the arrow keys
